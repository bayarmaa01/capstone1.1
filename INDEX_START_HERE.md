## 💬 COMMON VIVA QUESTION PATTERNS & HOW TO ANSWER

### Pattern 1: "Explain X"
```
Structure:
1. State what X is (1 sentence definition)
2. Explain how it works (2-3 sentences mechanism)
3. Give concrete example (show with numbers from paper)
4. Relate to system (why it matters for this project)

Example:
Q: "Explain cosine similarity"
A: "Cosine similarity measures the angle between two vectors, 
   ranging from -1 to +1. It's calculated as dot product divided 
   by magnitudes. For embeddings: S(xi,xj) = (xi·xj)/(||xi|| ||xj||). 
   In our system, if similarity > 0.70, we identify person as match. 
   We use it because it's fast and invariant to magnitude."
```

### Pattern 2: "Why is Y better than Z?"
```
Structure:
1. State what Y and Z are
2. Compare on key dimensions (accuracy, speed, privacy, simplicity)
3. Show numbers from table/figure if available
4. Conclude based on context

Example:
Q: "Why is CNN better than HAAR+SVM?"
A: "CNN achieves 97.8% accuracy vs. HAAR's 88.4%, a 9.4% improvement. 
   HAAR uses hand-designed features that can't handle lighting/pose 
   variations. CNN learns features from data (2,500 training images), 
   so it generalizes to real classroom conditions. Trade-off: CNN 
   slower (450ms vs. <100ms) but accuracy critical for institutional 
   fraud prevention."
```

### Pattern 3: "What if X happens?"
```
Structure:
1. Understand the hypothetical scenario
2. Trace through system logic
3. Identify where it would fail/succeed
4. Explain mitigations in place

Example:
Q: "What if system crashes?"
A: "System is designed for resilience. Database has automatic backups. 
   If recognition server crashes, QR fallback available. Infrastructure 
   defined as Terraform IaC, so we can redeploy in minutes. CI/CD pipeline 
   ensures deployment consistency. For long-term failure, disaster recovery 
   test: terraform apply, entire system rebuilt."
```

### Pattern 4: "How would you improve X?"
```
Structure:
1. Identify current limitation
2. Propose concrete improvement
3. Explain how it would help
4. Acknowledge trade-offs

Example:
Q: "How would you improve security?"
A: "Add liveness detection (Page 6 future work). Current system can't 
   distinguish real face from photo. Attacker could print John's photo 
   and scan it. Liveness detection: Require blinking, head movement, 
   or other anti-spoofing checks. Would reduce false positives from 0.9% 
   to <0.1% and prevent presentation attacks. Trade-off: Slight latency 
   increase and user interaction required."
```
